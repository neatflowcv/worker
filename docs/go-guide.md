# Go 가이드

## 목적

이 문서는 Go 코드를 변경한 뒤 반드시 수행해야 하는 최소 점검 절차를 정의합니다.

## 기본 규칙

- Go 코드를 작성하거나 수정했다면 아래 명령을 반드시 순서대로 실행합니다.
- 일부 명령만 실행하고 작업을 종료하면 안 됩니다.
- 모든 명령이 성공하기 전에는 작업을 완료로 보고하면 안 됩니다.
- 자동 수정으로 해결되지 않는 문제는 수동으로 고친 뒤 처음부터 다시 실행합니다.
- 메서드 안에서 `if x == nil { ... }`,
  `if len(x) == 0 { return nil }` 같은 방어 코드는 작성하지 않습니다.
- 인터페이스는 이를 사용하는 패키지와 별도의 패키지에 둡니다.
- 구현체는 인터페이스 패키지의 하위 패키지에 둡니다.
  예를 들어 인터페이스가 `foo/foo.go`에 있다면,
  구현체는 `foo/bar/foo.go`처럼 하위 패키지에 둡니다.
  즉, 같은 패키지명 아래에 인터페이스와 구현체를 함께 두지 않습니다.
- 구현체 하위 패키지에는 테스트를 작성하지 않습니다.

## 필수 점검 순서

```bash
golangci-lint fmt
golangci-lint run --fix
golangci-lint fmt
```

## 완료 조건

- 위 순서를 실제로 실행해 모두 통과해야 합니다.
- Go 테스트가 있는 변경이라면
  [docs/test-guide.md](docs/test-guide.md)의 원칙에 따라
  `go test ./...`까지 확인합니다.
