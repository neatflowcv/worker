# 마크다운 가이드

## 목적

이 문서는 이 저장소에서 마크다운 문서를 수정할 때 지켜야 하는 최소 규칙을 정리합니다.

## 기본 규칙

- 제목은 순서대로 `#`, `##`, `###`를 사용합니다.
- 헤더 앞뒤, 리스트 앞뒤, 코드 블록 앞뒤에는 빈 줄을 둡니다.
- 코드 블록에는 가능하면 언어 식별자를 붙입니다.
- 문서 링크는 저장소 기준 상대 경로를 사용합니다.
- 사용자 홈 디렉토리나 로컬 절대경로는 문서에 남기지 않습니다.

## 수정 후 검사

마크다운을 수정한 뒤에는 반드시 전체 마크다운 파일에 대해 린트를 실행합니다.

```bash
bunx -bun markdownlint-cli2 "**/*.md" "!vendor/**"
```

`vendor` 디렉토리는 검사 대상에서 제외합니다.

## 자동 수정

가능한 경우 자동 수정부터 시도하고, 남은 문제만 수동으로 고칩니다.
